## 시퀀스: 소셜 로그인 (Google/카카오)

```
사용자                 허브(tomatoeggcat.com)          소셜 제공자        Supabase DB
  │                          │                              │                  │
  │── /sign-in 클릭 ────────>│                              │                  │
  │                          │── 소셜 OAuth URL 생성 ──────>│                  │
  │<── 리디렉트 ─────────────│                              │                  │
  │                          │                              │                  │
  │── 소셜 로그인 동의 ──────────────────────────────────>│                  │
  │<── authorization_code ─────────────────────────────────│                  │
  │                          │                              │                  │
  │── /api/auth/callback ───>│                              │                  │
  │                          │── 토큰 교환 요청 ───────────>│                  │
  │                          │<── access_token + 프로필 ────│                  │
  │                          │── users.upsert() ────────────────────────────>│
  │                          │<── User 객체 ────────────────────────────────│
  │                          │── JWT 발급 (plan, credits 포함)               │
  │<── JWT + 쿠키 세팅 ──────│                              │                  │
  │                          │                              │                  │
  │── /dashboard 접근 ──────>│                              │                  │
  │                          │── JWT 검증 (stateless) ──────                  │
  │<── 대시보드 응답 ────────│                              │                  │
```

## 마이크로서비스 접근 시 JWT 전달 방식

```
사용자                 마이크로서비스              허브(선택적)
  │                        │                          │
  │── 요청 + JWT 쿠키 ────>│                          │
  │                        │── JWT 서명 검증 (로컬)   │
  │                        │   (허브 공개키 또는       │
  │                        │    shared JWT_SECRET)     │
  │                        │                          │
  │                        │ [plan/credits 체크]       │
  │                        │   충분 → 서비스 응답      │
  │                        │   부족 → 403 + 허브       │
  │                        │          /pricing 리디렉트 │
  │<── 서비스 응답 ─────────│                          │
```

## 쿠키 전략

### 서브도메인 통일 시 (권장)

```
Set-Cookie: token=<JWT>; Domain=.tomatoeggcat.com; HttpOnly; Secure; SameSite=Lax
```

모든 `*.tomatoeggcat.com` 에서 자동 전달.

### 별도 도메인 마이크로서비스 시

- 허브에서 JWT를 URL 파라미터 또는 postMessage로 전달
- 마이크로서비스가 로컬 스토리지에 저장 (HttpOnly 불가, XSS 주의)
- 또는 각 서비스에서 허브 API를 호출해 토큰 검증 (네트워크 비용 발생)

## AdSense 공개 페이지 처리

```
사용자 (비로그인)         마이크로서비스
  │                           │
  │── 페이지 요청 ───────────>│
  │                           │── allowGuest: true 미들웨어
  │                           │── 광고 표시 활성화
  │<── 공개 컨텐츠 + 광고 ───│
```

```
사용자 (로그인 + pro)      마이크로서비스
  │                           │
  │── 페이지 요청 + JWT ─────>│
  │                           │── JWT plan: 'pro' 확인
  │                           │── 광고 표시 비활성화
  │<── 프리미엄 컨텐츠 ───────│
```
