## 크레딧 충전 — 토스페이먼츠 단건 결제

```

  │                      │                       │                       │
  │── /credits 방문 ────>│                       │                       │
  │<── 충전 옵션 표시 ────│                       │                       │
  │   (10크레딧 = 1,000원 등)                     │                       │
  │                      │                       │                       │
  │── "충전" 클릭 ───────>│                       │                       │
  │                      │── 토스 결제 위젯 로드  │                       │
  │<── 결제창 표시 ────── │                       │                       │
  │                      │                       │                       │
  │── 결제 완료 ─────────>│                       │                       │
  │                      │── POST /api/billing/toss-confirm              │
  │                      │   { paymentKey, orderId, amount } ────────────>│
  │                      │                       │── 토스 결제 승인 API   │
  │                      │                       │── credits += N ───────>│
  │                      │                       │── credit_transactions  │
  │                      │                       │   INSERT ─────────────>│
  │                      │<── { credits: 새잔액 } │                       │
  │<── "크레딧 충전 완료" │                       │                       │
  │   잔액 업데이트 표시 ─│                       │                       │
```

  │                 │── JWT 파싱 → credits 확인 │                       │
  │                 │                          │                       │

  │                 │                          │                       │
  │                 │ [credits >= 필요량]       │                       │

  │                 │                          │── SELECT FOR UPDATE    │

## 크레딧 원자성 보장 설계

크레딧 이중 차감을 방지하려면 DB 레벨 원자적 업데이트 필요.

```sql
-- Supabase RPC 또는 서버 함수로 구현 권장
-- 음수 방지 + 트랜잭션 보장
UPDATE users
SET credits = credits - :amount,
    updated_at = NOW()
WHERE id = :userId
  AND credits >= :amount   -- 잔액 부족 시 업데이트 안 됨
RETURNING credits;
-- affected rows = 0 이면 InsufficientCreditsError 반환
```

## 크레딧 가격 정책 예시 (설계 참고)

| 패키지 | 크레딧 | 가격 (KRW) | 단가 |
|---|---|---|---|
| 소 | 10 | 1,000원 | 100원/크레딧 |
| 중 | 50 | 4,500원 | 90원/크레딧 |
| 대 | 100 | 8,000원 | 80원/크레딧 |

Pro 구독자는 매월 크레딧 N개 자동 지급 + 추가 충전 가능으로 설계 가능.

## 크레딧 vs 구독 병행 전략

```
사용자 유형            권장 플랜
─────────────────────────────────
비로그인              AdSense 광고 표시 (무료)
free 로그인           광고 표시 + 일부 무료 기능
pro 구독             광고 제거 + 무제한 기능 + 월 크레딧 포함
credits_only          광고 표시 + 크레딧으로 프리미엄 기능 단건 이용
```

## 환불/만료 정책 (설계 메모)

- **유료 크레딧**: 구매일로부터 1년 유효 (선택적, 법률 확인 필요)
- **구독 포함 크레딧**: 구독 기간 내 사용, 미사용분 소멸 또는 이월 (정책 결정 필요)
- **환불**: 미사용 크레딧 환불 시 credit_transactions 역산 + 결제사 환불 API 호출
- **실제 환불 처리는 사용자(운영자) 본인이 결제사 대시보드 또는 API로 직접 수행**
