# 정상인가요 — 콘텐츠 보이스 & 게이트 (P2 작성 기준)

시드 설문·카테고리 허브·주간 통계 해설 모두 이 기준을 따른다. 적대적 심사관(P6)이 이 기준으로 탈락시킨다.

## 보이스

- 친구가 옆에서 말하듯. 건조한 백과사전 톤 금지, 과한 개그 톤도 금지.
- 판단하지 않는다. "그게 정상이다/비정상이다"를 단정하지 않고 "몇 퍼센트인지"만 말한다.
- 통계 숫자를 지어내지 않는다. 시드엔 아직 표본이 없다 → 해설은 "왜 갈리는지"를 다루되
  구체 퍼센트는 절대 쓰지 않는다("응답자의 70%가…" 류 금지). 실제 집계는 배포 후 채워진다.

## 절대 금지 (검출 시 재작성)

- AI 상투구: 알아보겠습니다 / 살펴보겠습니다 / 결론적으로 / 중요합니다 / 도움이 되셨길 / 마무리하며
- 지어낸 통계·퍼센트·"연구에 따르면"류 출처 없는 수치
- 의료 진단/처방 (sleep·body·mind 카테고리 특히): "불면증입니다", "이건 병입니다" 금지.
  증상 나열·자가진단 유도 금지. 오직 습관·감각·성향의 "빈도 비교"만.
- 투자 권유/수익 보장 (money): "이렇게 투자하세요" 금지. 소비 습관 비교만.
- 특정 개인 식별: "저희 팀장" OK / "○○기업 김○○ 팀장" 금지. 실명·상호·연락처 삽입 금지.
- 도박/성인/혐오.

## 다양성 강제 (F8/A3/C4 대비)

- 12개 허브 intro가 같은 문장 구조로 시작하면 실패. 오프닝을 서로 다르게:
  질문형 / 장면 묘사 / 통념 반박 / 상황 나열 / 고백 톤 등 섞는다.
- 60개 시드 title이 "○○ 하나요?"로만 끝나면 단조롭다. 어투를 섞되 전부 답 가능한 질문형.
- 시드 body/commentary가 문장구조 80% 이상 겹치면 병합·재작성 대상.

## 시드 설문 스키마 (JSON)

```jsonc
{
  "slug": "towel-reuse-days",          // 영문 kebab, 고유
  "categorySlug": "living",
  "title": "수건 한 장, 며칠 쓰고 빨래통에 넣으세요?",   // 답 가능한 질문
  "body": "샤워하고 딱 몸만 닦는데 매번 새 수건? ...",     // 120~250자, 긴장 세팅, PII·의료단정 없음
  "options": [                          // binary 기본 (2개). 자연스러운 라벨.
    {"key": "a", "label": "하루 쓰고 바로 빤다"},
    {"key": "b", "label": "2~3일은 다시 쓴다"}
  ],
  "editorCommentary": "이 질문이 갈리는 진짜 이유는 ..."   // 300~420자, 이 설문에만 있는 고유 텍스트
}
```

- options는 2지선다 기본. 애매하면 "그렇다/아니다"보다 구체 행동 라벨이 낫다(참여율↑).
- editorCommentary는 "왜 사람마다 다른가 / 어떤 배경에서 갈리나 / 소수파는 어떤 사람인가"를
  구체적으로. 뻔한 일반론 금지. 퍼센트 지어내기 금지.

## 카테고리 허브 스키마 (JSON)

```jsonc
{
  "categorySlug": "living",
  "h1": "생활·집안습관, 다들 어떻게 살까",
  "metaTitle": "...",                 // 55자 내외, 카테고리마다 고유
  "metaDescription": "...",           // 150자 내외, 고유
  "intro": "...",                     // 1,200자+ 고유 편집 본문. 이 카테고리의 '이거 나만 그래' 지형을 그린다
  "faq": [                            // 3~4개. FAQPage 스키마용
    {"q": "...", "a": "..."}          // a는 각 120자+ 고유
  ]
}
```

- intro 1,200자+ 전부 이 카테고리 전용. 변수 치환·공통 문구로 채우면 고유 텍스트로 안 셈.
- 목록/숫자 나열로 글자수 때우기 금지. 실제 생활 장면·논쟁 지점을 서술.
