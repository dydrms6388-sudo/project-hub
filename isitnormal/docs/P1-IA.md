# 정상인가요 (isitnormal) — P1 정보구조(IA) 확정

> "우리집만 이래?"를 익명 투표로 1탭에 해소하는 통계 커뮤니티.
> 이 문서는 P1 산출물의 서술 버전이다. 기계가 읽는 실제 정의는
> `content/categories.ts`, `content/card-copy.ts`, `lib/promotion-gate.ts`,
> `supabase/schema.sql` 에 있으며 이 문서는 그 네 파일을 묶는 설계 근거다.

---

## 0. 배포·기술 결정 (근거)

- 이 리포는 정적 HTML 팩토리(225개)와, 백엔드가 필요한 **독립 앱**(예: `auth-hub/`,
  자체 `package.json`, 자체 Vercel 프로젝트, 루트 `vercel.json` rewrites 프록시)이
  공존한다. isitnormal은 투표/RLS/모더레이션/서버 집계가 필수라 정적 HTML로는 불가능하다.
- 따라서 **독립 Next.js App Router + TS + Tailwind + Supabase** 앱을 `isitnormal/` 에 두고,
  자체 Vercel 프로젝트로 배포한 뒤 허브에서 rewrite 프록시한다(auth-hub와 동일 패턴).
- 도메인: tomatoeggcat.com은 "Low Value Content"로 이미 거절된 전례가 있으므로,
  isitnormal은 **별도 서브도메인/도메인**으로 심사받는다(예: `isitnormal.*`).
  이 결정으로 기존 거절 이력과 색인 오염을 분리한다.

## 1. 서비스 정체 — 엔진

도구가 아니라 UGC 플랫폼이다. 콘텐츠는 유저가 쓴다. 우리 일은 "쓰게 만드는 구조".

```
투표(획득) → 통계(콘텐츠) → 알림(리텐션)
```

참여 계단(강제 금지):

| 단계 | 비용 | 로그인 | 이탈 지점 |
|---|---|---|---|
| 투표 | 1탭 | 없음 | 90%가 여기서 끝. 정상이다. |
| 한 줄 의견 | 10초 | 없음 | 선택 |
| 내 설문 제작(UGC) | 2분, 400자+ | 없음 | P5에서 개방 (투표 트래픽 생긴 뒤) |

**핵심 규칙:** 투표 전엔 결과를 절대 안 보여준다(참여율 3배). 단 "결과만 보기" 링크는 항상 제공.

## 2. 뿌릴 이유 (바이럴 명제)

> 유저는 **내가 소수파라는 게 신기해서** 스스로 링크를 뿌린다.

"공유해주세요" 버튼이 아니라, 결과 카드가 "나에 대한 발견"이라서 자발적으로 퍼진다.
카드 명제 = "당신은 상위 N% 소수파" (해석 문구 12종은 `content/card-copy.ts`).

## 3. 콘텐츠 축

```
카테고리(12) × 설문(무한) × 응답자 속성 교차 통계
```

- **카테고리 12개**: 일상 "이거 나만 그래?"가 실제로 발생하는 생활 영역. `content/categories.ts`.
- **설문(survey)** = 콘텐츠 원자 단위. "○○ 하는 사람?" 이분 투표(그렇다/아니다) 기본.
- **교차 통계**: 응답자가 선택 입력한 속성(연령대/성별/지역)으로 크로스탭.
  속성은 전부 자기신고·선택·마스킹 대상 아님(식별 불가 밴드만). 미입력이 기본값.

## 4. 색인 자산 3종 (이 3종만 색인한다)

| 자산 | 색인 시점 | 고유 텍스트 출처 |
|---|---|---|
| 승격 설문 페이지 | 게이트 통과 후 | 편집자 해설 300자+ (설문마다 다름) |
| 주간 통계 해설 | 매주 발행 | 해설 1,200자 (매주 다름) |
| 카테고리 허브 12개 | 발행 즉시(편집 콘텐츠) | 허브 도입부·FAQ 1,200자+ (허브마다 다름) |

- 카테고리 허브는 UGC가 아니라 **운영자 편집 콘텐츠**라 첫날부터 색인 가능(원문 게이트 무관).
  단 허브당 고유 본문 1,200자+ / 그중 800자+ 페이지 전용을 충족해야 한다.
- 승격 설문·주간 통계는 실제 투표 n≥30이 쌓인 뒤에만 통계 노출·색인. **가짜 숫자 금지.**

## 5. 승격 색인 게이트 (최우선 원칙 1)

기본값: **모든 UGC는 noindex.** 미승격 UGC는 sitemap 절대 금지.
색인 페이지 "수"가 아니라 "밀도"다. 로직 원본은 `lib/promotion-gate.ts`.

승격(index 승격) 필수 조건 — **전부** 충족:

1. 본문 400자+ **또는** 투표 30표+
2. 유효 반응(투표/댓글) 10개+
3. 작성 후 7일 경과
4. 자동 품질 스캔 통과 (상투구·유사도·법무·스팸)

그리고 승격 페이지는 3층 구조를 합쳐 **1,200자 초과**:

```
[1층] UGC 원문
[2층] 투표 통계 시각화 (n≥30일 때만; 아니면 "집계 중")
[3층] 편집자 해설 300자+  ← 이 설문에만 존재하는 고유 텍스트
[+]   관련 항목 5개
```

- 3층 중 하나라도 없으면 noindex (절대 규칙 1).
- 고유 텍스트 800자+ (변수 치환 템플릿 문장은 고유로 안 셈, 절대 규칙 2).
- 운영자 시드 설문도 동일 게이트. 단 원문·해설은 작성 시점부터 완비되므로 실질 병목은
  "실제 투표 n≥30"이다. **시드도 30표 전엔 noindex + 집계 중.** (S2/D3)

sitemap = 승격 설문 + 주간 통계 + 카테고리 허브 + 고정 페이지. 그 외 전부 제외.

## 6. 콜드스타트 (유저 0명 생존법)

- 오픈 시 시드 설문 **최소 60개** 직접 작성, 전부 `@운영자` 배지. 사용자인 척 금지(S1).
- n<30 항목은 "집계 중" 표기. 가짜 숫자 절대 금지(S2).
- 항목 3개 미만 섹션은 섹션 자체를 숨기고 단일 CTA만(S3).
- 첫 2주 신규 UGC는 홈에 강제 노출(반응 0 방지, S4).
- v1(P1~P4)은 투표만 개방. UGC 작성(P5)은 투표 트래픽 확인 후 개방.

## 7. 법무 게이트 (오픈 전 필수)

| 코드 | 규칙 | 구현 위치(예정) |
|---|---|---|
| L1 | 실명/상호/직장/학교 자동 검출 → 마스킹 후 게시, **원문 저장 금지** | insert 전 스캐너 (P5) |
| L2 | 전화/계좌/주소/차량번호 패턴 → 즉시 반려 | insert 전 스캐너 (P5) |
| L3 | 특정 개인 식별 사연 → 반려 ("저희 팀장" OK / "○○기업 김○○ 팀장" 반려) | 스캐너+휴먼 큐 |
| L4 | /takedown 48시간 내 처리 명시 + 실제 처리 큐 | `takedown_requests` 테이블 |
| L5 | 전 UGC 하단 고정 고지: "이용자 작성 콘텐츠이며 운영자는 진위를 보증하지 않습니다" | 레이아웃 |
| L6 | 이미지 업로드 금지. 텍스트 + 이모지만 | 업로드 UI 부재로 원천 차단 |

## 8. 어뷰징 방어

| 코드 | 규칙 |
|---|---|
| V1 | 투표 중복: 세션+IP+fingerprint 3중. 24시간 재투표 금지 |
| V2 | 신규 항목 급등 감지 → 자동 홀드 후 검토 (`status='held'`) |
| V3 | 통계 페이지 표본수 병기. n<30이면 통계 미노출, "집계 중" |
| V4 | 봇 투표 비율 추정치를 관리자 대시보드 노출 |
| V5 | 익명 rate limit: IP+세션당 1일 작성 3건 / 투표 50건 |

## 10. 공유 URL 정책

- 공유 링크 `/s/{shortId}`. robots.txt Disallow + noindex 전면.
- OG 태그만 살림(og:image=카드, og:title=해석 문구, og:description=참여 유도).
- 링크 열면 결과 미리보기 → "나도 투표하기" CTA → 색인된 본 페이지로.
- 결과는 URL 파라미터가 아니라 shortId 서버 조회(파라미터 무한조합 색인 차단). shortId 90일 만료.
- robots Disallow: `/s/` `/api/` `/pending/` `/vote/`.

## 11. 계측(K값 추적)

이벤트: `view` `vote` `result_view` `card_render` `share_click` `share_landing`
`share_to_vote` `ugc_submit` `notify_optin`.
K값 = 공유당 신규 유입 = `share_landing→(vote|view)` 전환 / `share_click`.

## 12. 목표

색인 200+ / 시드 60 / 4주차 K값 0.3+.

## 13. JSON-LD 규격

- 승격 설문 = `Article`(미승인이면 제외) + `BreadcrumbList` + `FAQPage`(해당 시).
- 통계 페이지 = `Dataset`.
- **미검증 사연에 Review/AggregateRating 절대 금지**(구조화 데이터 스팸).
